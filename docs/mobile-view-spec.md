# ТЗ: мобильный вид

1) Макет/брейкпоинты  
- Мобильный брейкпоинт ≤ 640px.  
- Сайдбар скрыт; вместо него — кнопка в шапке («Статьи»/гамбургер) открывает выезжающую панель/модал с поиском и списком.  
- Основной контент на всю ширину экрана с отступами ~12–16px, без горизонтальной прокрутки.

2) Блоки и DnD  
- Высота интерактивных кнопок (свернуть/drag) ≥ 36–40px, увеличенная зона клика.  
- Кнопка сворачивания слева от текста; drag‑handle фиксирован справа по одной вертикали, вне зависимости от вложенности.  
- Синяя линия вставки тянется на всю ширину блока и остаётся внутри его границ; выход в родителя определяется жестом вверх/вниз или по левому порогу.

3) Режим редактирования  
- Текстовое поле на всю ширину блока, лёгкая рамка/подсветка без «капсулы».  
- Отдельная панель действий под блоком (высота ~44–48px): «Отмена» слева, «Сохранить» справа; кнопки округлые, сильно разнесены.  
- Вложенные блоки при редактировании не смещают родителя; кнопки остаются видимыми.

4) Шапка/управление  
- Заголовок статьи + кнопка меню (⋮) в шапке; время обновления — в одну строку или малым текстом под заголовком.  
- Кнопка открытия списка статей в шапке (гамбургер/«Статьи»).

5) Стили  
- Отступы блоков ~10–12px по вертикали, минимальные тени, светлый фон; шрифты 15–16px, line-height 1.4–1.5.  
- Убираем лишние капсулы, оставляем тонкие обводки для активных элементов.

6) Проверки  
- Добавить мобильный снап (например, viewport 375x667) и прогонять `npm run snap` с этим размером.  
- В отчётах по UI: проверять PNG — хэндлы на правой вертикали, кнопки edit разнесены снизу, полоска вставки внутри блока.
