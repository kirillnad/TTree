# UI checklist (напоминание для себя)

- После любых изменений внешнего вида **обязательно** прогонять `npm run snap` и открывать свежий `snapshots/drag-handles.png` глазами — не ограничиваться JSON-координатами.
- Не пользоваться старым снапом: если обсуждаем UI, сначала сгенерировать новый `npm run snap` и смотреть **текущий** PNG, затем делать выводы.
- При обсуждении визуальных деталей (позиция/видимость кнопок, цветов) **всегда** сверяться со скриншотом и включать описание увиденного; не надеяться только на метрики.
- Правая граница всех блоков должна совпадать: при любой вложенности блоки заканчиваются на одной вертикальной линии. Это правило не нарушаем.
- Когда блок редактируется, его дочерние блоки не остаются внутри `.block.selected.editing`, а рисуются ниже — иначе выделение и сусцеестветсвующие хэндлы могут смещаться.
- Кнопки DnD (хэндлы) видны у каждого блока, даже если у него нет заголовка; их положение справа фиксировано и не исчезает.
- Проверять на снапе:
  - Хэндлы DnD каждого уровня: нет ли наложений, все ли в своей строке.
  - Контраст/цвет иконок, читаемость кнопок и текста.
  - Отступы и границы: не “плавают” ли блоки, не съезжают ли линии вложенности.
- Только после визуальной проверки писать, что всё ок; не перекладывать просмотр на пользователя.
